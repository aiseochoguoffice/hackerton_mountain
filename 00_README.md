## 데이터 기반 핵심 인사이트 (소방청 CSV 분석)

### 사고 유형 분류
```
실족추락  ████████████████████ 55%  ← 가장 많음
개인질환  ████████             24%
일반조난  █████                15%
기타산악  ██                    6%
```

### 위험 지수 계산 공식
```
위험지수 (0~100) =
  사고 빈도 가중치    40% (전국 산 대비 상대값)
+ 중증도 가중치       30% (항공대인계 비율)
+ 계절 보정          15% (봄·가을 단풍 시즌 +)
+ 시간대 보정        15% (14~16시 하산 집중 +)

LOW: 0~30 | MEDIUM: 31~60 | HIGH: 61~80 | VERY_HIGH: 81~100
```

### 핵심 사용자 가치
1. **사전 예방**: 산행 전 체크리스트로 준비 완료도 점검
2. **정보 비대칭 해소**: "이 산에서 어떤 사고가 많이 나는지" 데이터로 확인
3. **개인화 조언**: 체크 결과 기반 맞춤형 경고 & 팁 제공

## 발표 포인트 (해커톤 심사 기준 대응)

| 심사 항목 | 대응 내용 |
|---------|---------|
| **문제 정의** | 소방청 데이터로 확인된 연간 산악사고 현황, 사전 예방 정보 부재 |
| **데이터 활용** | 소방청 공공데이터 2개 파일 → 위험 지수 계산 알고리즘 |
| **실용성** | 체크리스트 12문항 → 즉각적인 산행 안전 점검 가능 |
| **확장성** | DB 설계 + API 명세 + 날씨 연동 포인트 명시 |
| **차별성** | 단순 통계 보여주기가 아닌, 개인화된 체크리스트 → 맞춤 경고 UX |
